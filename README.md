# E-Commerce SQL Data Analysis Project

This project is an end-to-end SQL-based data analysis performed on an e-commerce dataset.  
The objective of this project is to analyze business performance, customer behavior, sales trends, payment patterns, delivery efficiency, and advanced analytical metrics using SQL.

This project is created to demonstrate practical data analyst skills and real-world business problem solving using SQL.

---

## Tools Used
- SQL Server
- SQL (Joins, CTEs, Aggregations, Subqueries, Window Functions)
- GitHub

---

## Project Structure
The analysis is divided into multiple logical sections, each focusing on a specific business area:

- Platform Overview  
- Customer Analysis 
- Repeat Customer Analysis 
- Sales and Revenue Analysis  
- Product and Category Analysis  
- Payment Analysis  
- Delivery Performance Analysis    
- Advanced Analytics and Business Insights  

Each section is written as a separate SQL file for better readability and modular analysis.

---

## Dataset Information
The dataset used in this project is a public e-commerce dataset containing information related to:
- Orders
- Customers
- Products
- Sellers
- Payments
- Reviews
- Delivery details

## Dataset Source  
Public e-commerce dataset sourced from Kaggle and used for learning and analysis purposes.
https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce

---

## Key Business Insights
- Sales and revenue are unevenly distributed across regions and product categories
- Certain payment methods dominate customer transactions
- Delivery delays have a direct impact on customer satisfaction
- Repeat customers contribute higher average revenue compared to one-time customers
- Customer review scores are strongly influenced by delivery performance
